## Estructura de un Subagente en Deepagents

Para organizar tus subagentes en un archivo `subagents.py`, puedes definir una lista de objetos `SubAgent` (que son `TypedDict`). Cada subagente requiere un `name`, `description` y `system_prompt`.

### Ejemplo de `subagents.py`
```python
from deepagents.middleware.subagents import SubAgent

# Definición de subagentes declarativos
RESEACHER_AGENT: SubAgent = {
    "name": "researcher",
    "description": "Agente especializado en búsqueda y síntesis de información.",
    "system_prompt": "Eres un investigador experto. Usa las herramientas de búsqueda para responder.",
    "model": "openai:gpt-4o", # Opcional: sobreescribe el modelo del padre
    "tools": [] # Opcional: herramientas específicas
}
```

## Integración en el Agente Principal
Una vez definidos, impórtalos y pásalos al argumento `subagents` de `create_deep_agent`. El sistema registrará automáticamente la herramienta `task` para que el agente principal pueda delegarles trabajo.

```python
from deepagents import create_deep_agent
from subagents import RESEACHER_AGENT

agent = create_deep_agent(
    model="openai:gpt-4o",
    subagents=[RESEACHER_AGENT]
)
```

## Tipos de Subagentes
| Tipo | Definición | Uso Principal |
| :--- | :--- | :--- |
| **Declarativo (`SubAgent`)** | Diccionario con instrucciones y herramientas. | Definición rápida de agentes efímeros. |
| **Compilado (`CompiledSubAgent`)** | Usa un objeto `runnable` pre-construido. | Cuando necesitas un grafo de LangGraph complejo. |
| **Asíncrono (`AsyncSubAgent`)** | Ejecución remota mediante `graph_id`. | Tareas de larga duración en segundo plano. |

## Notas
- **Herramienta `task`**: Es el mecanismo que usa el LLM para invocar a los subagentes sincronos.
- **Aislamiento**: Cada subagente tiene su propia ventana de contexto, lo que ayuda a ahorrar tokens y reducir ruido en el hilo principal.
- **General Purpose**: Por defecto, se incluye un agente `general-purpose` a menos que se desactive en el perfil.
